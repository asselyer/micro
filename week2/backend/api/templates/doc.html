How it works

CategorySerializers uses Serializers which associates the Model object with an easily processible data type. In this case, it associates id of products category with the name of products category UserSerializer uses a ModelSerializer for the user so that a programmer can easily access the model of the user (not concerning about the deal with Model instances and querysets ). Additionally, it provides a tool for validation of user passwords

Note danger:Category class stores the categories of the products. It invokes CategoryManager() which sets the display format in website’s administration. URL patterns associate the url with appropriate function in views. For example, when the server requests localhost/login, it will invoke views.login function, which will act appropriately.

Info

At homepage, we can see list of users, products, and orders. The User ‘class’ stores information about user such as its user name and email. Similarly, Product ‘class’ contains information about the name of the product, quantity, price, and Order ‘class’ informs about the products being purchased, status of the payment, etc
